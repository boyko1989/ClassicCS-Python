# Простейшее сжатие

Это всегда компромисс между пространством и временем: если распаковывать дольше, чем позволяет экономическая целесообразность от сэкономленного времени, то и сжимать не стоит.

Сжатие может сводиться к тому, что информация представляется не в виде стандартных типов данных, а в виде того количества битов, которое необходимо для определения некоторого количества уникальных значений сохраняемых данных.

Тут предлагается зашифровать 4 нуклеотида входящих в состав ДНК двумя битами: их же 4, значит и битов для каждого достаточно двух. В этой книге к ДНК вообще какая-то неприкрытая любо**В**...

В методе **ComperssedGene._compress**, который вызывается в конструкторе, мы перекодируем информацию причём по следующему алгоритму:
+ Создаём объект `ComperssedGene` передавая ему интересующую строку
+ При создании объект делает следующее:
  + Создаёт последовательность бит (в начале там всего лишь единица) `bit_string = 1`
  + Проходится по каждому символу в строке `for nucleotide in gene: ...`
+ Сдвигом добавляем в `bit_string` два пустых бита
+ Условиями проверяем как должны быть заполнены два новых бита
+ "ИЛькаем" нашу битовую последовательность с высчитанным кодом, чтобы этот код добавить в последовательность `bit_string`

Ну и методом **ComperssedGene.decompress** мы формируем новую строку из юитовой последовательности. Работаем уже с обратной стороны последовательности, сдвигая фокус внимания программы с правого края `bit_string` по два бита. Проверяем их по кодам нуклеотидов и возвращаем строку в обратном отображении (`[::-1]`)
